# Review Checklist

Use this after a design has been selected or implemented.

## Accessibility

- Semantic landmarks are correct.
- Headings follow a logical order.
- Images have meaningful alt text.
- Interactive controls have labels.
- Keyboard navigation works end to end.
- Focus states are visible.
- Color is not the only way status is communicated.

## Interaction

- Clickable items look clickable.
- Hover, press, and disabled states are distinct.
- Forms show visible labels and actionable errors.
- Loading states are present for async actions.
- Destructive actions are separated from primary actions.

## Responsive

- Layout starts from the smallest screen.
- No horizontal scroll on mobile.
- Typography stays readable across breakpoints.
- Fixed bars and overlays do not hide content.
- Tablet and desktop spacing stay consistent.

## Performance

- Heavy media is optimized and sized.
- Layout shifts are avoided.
- Animations stay on transform and opacity where possible.
- Below-the-fold content does not block initial rendering.
- Third-party code is justified, not defaulted.

## Visual Quality

- The page has a clear hierarchy.
- Spacing is consistent.
- Color usage is intentional and limited.
- Icons come from one visual language.
- The result does not feel templated or generic.
